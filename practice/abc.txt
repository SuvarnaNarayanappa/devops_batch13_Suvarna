Link

A link is a reference/pointer to a file/dir
Link allows multiple access points to same data without duplicating files

Types:1`

1. Hardlink

Hard link is an additional name for existing file where both the original file and hard link
will share the same inode which means they are same file with diff names

inode (index node): It is data structure that stores information about file/dir's size,owner..etc

synatx: ln <file-name> <link-name>

Note:

1. When you make any changes to original file,we can see those changes in the link
2. If you rename the original file,the link will work
3. If we delete the original file then also your link will work

2. Softlink

A softlink is a pointer to a file. It does not share same inode as the original file


synatx: ln -s <file-name> <link-name>

Note:

1. When you make any changes to original file,we can see those changes in the link
2. If you rename the original file,then link will not work
3. If we delete the original file then link will not work
